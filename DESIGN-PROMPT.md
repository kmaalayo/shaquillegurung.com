CONCEPT — "THE HONEST ENGINE"
================================================================
An editorial laboratory: a physicist's annotated notebook crossed with a Swiss clinical-data report — the look of an instrument that measures honestly. Confidence is the entire visual language. Certainty reads as crisp, opaque, still ink; uncertainty reads as translucent, faintly trembling, lighter-weight ink — so the page's visual mood literally encodes how sure it is. Anything the MACHINE measures (a confidence %, a source URL, a latency, a timestamp) speaks in monospace; anything the FOUNDER asserts speaks in serif. You can always tell which voice is which.

THE SIGNATURE MOMENT — a two-part hero so the thesis lands instantly AND pays off on interaction.

PART A (instant, ≤3s) — PROXIMITY TYPE THESIS.
At the very top, one founder-voiced line set huge in Fraunces: **"I build AI that tells you when it doesn't know."** Split it per glyph (wrap each glyph in a `<span>` with `aria-hidden` on the decorative spans and keep the full sentence available to screen readers as a single accessible label). On pointermove, each glyph reads its distance to the cursor and animates its variable-font axes by proximity — wght 300→820, opsz tracking rendered size, SOFT 0→100 — so letters near the cursor get heavier, softer, rounder while distant ones stay thin and crisp: a loupe of attention sweeping the words like a lens over newsprint. The cluster **"doesn't know"** is the accent: Fraunces italic in the accent color, and the ONLY cluster that also nudges +6px on the y-axis toward the cursor, so it feels physically reached-for. Pure `font-variation-settings`, no transforms warping shapes — GPU-cheap. CRITICAL PERF: do NOT recompute on every raw pointermove event — store the latest pointer coords in a variable on `pointermove` and update all glyph axes inside ONE shared `requestAnimationFrame` loop (single read/write pass, no per-glyph layout reads in the loop), so 60fps holds with ~30 glyphs. The type asserts itself loudly exactly where attention is and stays quiet, thin, honest everywhere else: confident only where it has grounds to be. Mono cue beneath: **"move your cursor — the type responds."** pointerleave eases all glyphs back to one tasteful resting weight.

PART B (the proof, ~20–30s) — THE PERTURBATION / live reasoning canvas, in the wide column directly below the thesis.
A real question is already typed into an input strip: **"Can a parent see another school's attendance?"** Below it the engine answers token-by-token. Build tokens as inline `<span>`s in the DOM — NOT a canvas — so they're accessible, selectable, crisp. Each output token fades+rises (y:8→0) at ~45ms stagger, but its OPACITY and a faint ±1px vertical sine wobble bind to a per-token confidence value baked into the markup (e.g. `data-conf="0.94"`). High-confidence words land crisp and still; low-confidence words arrive at ~40% opacity, breathe, AND animate their Fraunces wght DOWN (340→200) — uncertainty literally makes the type lighter. Factual claims draw a thin SVG connector (stroke-dashoffset draw-on) down to a hairline source card that slides in beside them (e.g. `source: noxnoctech.com · Postgres FORCE ROW LEVEL SECURITY · verified`).

Then invite the visitor to break it, in plain words near the control: **"Take away the evidence and watch it get more honest, not more confident."** A single button, **"remove the source,"** pulls the cited source card out. The engine RE-THINKS from ONE shared state value: an aggregate confidence meter (a thin rule with an accent fill) visibly drops; the now-unsupported span strikes itself through via an eased clip-path inset wipe (0.45s) and rewrites in place to **"I can't verify this without the source."** in the warning tone; the wobble on uncertain tokens intensifies. A **"/Why?"** toggle reveals the raw per-token confidence numbers in mono beside each token (e.g. `0.94`, `0.41`) — the mechanism made legible, including the wght value as a low-confidence token drops 340→200. You SEE the system showing its settings instead of hiding them. The button must be a real toggle: pressing it again restores the source and re-supports the claim, so the visitor can play the beat both ways. This interaction is impossible to fake with a screenshot — that is the explicit virality goal; tell the visitor it is a thing to DO.
